原型链

每个对象都有原型,原型也是一个对象,所以原型也有自己的原型,这样就形成了一个以圆形组成的链条,称为原型链

  • 属性搜索原则
    当一个对象去访问对象的属性的时候,遵守如下原则:
  1. 先在对象本身中去访问对象的属性,找到就直接使用
  2. 如果没有找到,就去原型中找,找到直接使用
  3. 如果没有找到,就继续沿着原型链向上查找,直到null为止

object.prototype成员介绍

Function的解释

Function是一个构造函数,可以用来创建函数对象
js中所有的函数(包括Function本身),都可以看做是使用Function构造函数创建出来的

Function可以用来创建函数

  1. 如果不给Function传递参数,那么创建出来的就是一个空函数

  2. 传递一个参数,这个参数就是函数的函数体,这个参数是字符串类型的

  3. 传递多个参数的时候,最后一个参数为函数体,之前的所有参数都是形参名

eval函数

eval是一个函数,可以把字符串当做代码来执行
和Function的区别在于
eval会将字符串转换成代码,并且直接执行
而Function创建出来的是函数,必须要手动调用,里面的代码才会执行

JSON格式字符串转换成JS对象
JSON.parse

使用eval来解析JSON格式的字符串
当使用eval直接解析json格式的字符串的时候，系统会将{}当做代码块来处理，而不是对象字面量，所以会报错！

代码块?
代码块只能单独存在,但是对象字面量可以作为橘子的一部分存在,给变量赋值或者传参

Function和eval 都存在安全性问题

函数的原型链

  1. 函数也是一个对象

instanceof 关键字

格式:
对象 instanceof 构造函数

函数的成员

  1. name : 函数的名称
  2. length: 函数的形参的个数
  3. caller： 函数是在哪个函数中被调用的，caller就指向那个外部函数（这个caller必须要在函数中去使用，才能访问到值！）
  4. arguments： 函数在被调用的时候，会把所有的实参存到这个对象里面（这个对象必须要在函数中去使用，才能访问到值！ 这个arguments是函数的属性，和之前学过的arguments不一样！）

静态成员和实例成员

  • 静态成员： 通过构造函数访问的成员就是静态成员
  • 实例成员： 通过实例对象访问的成员就是实例成员
