arguments对象

arguments对象就是一个伪数组，他是函数中的一个变量
当函数被调用的时候，传入函数的所有的实参，都会被存入arguments这个伪数组中

一般在用户传入的参数个数不确定的情况下，就使用arguments来获取用户传入的参数，而不通过形参来获取

arguments对象的属性
callee: 指的就是arguments对象所在的函数! 一般用在函数中实现递归

length: 指的是用户在调用函数的时候,传入的实参的个数

递归(算法)

在函数中,直接或者间接的调用自己

递归的两大要素

  1. 必须要有自己调用自己
  2. 必须要有结束条件

化归思想

化归思想，将一个问题由难化易，由繁化简，由复杂化简单的过程称为化归，它是转化和归结的简称。

词法作用域

作用域

变量起作用的范围,就是变量的作用域

在JS当中有且只有函数可以创建作用域

全局作用域：直接写在script标签中的内容，就属于全局作用域的范围
局部作用局：写在函数中的内容，属于局部作用域的范围
块级作用域: （JS中没有块级作用域）通过代码块限定的作用域，就是块级作用域

词法作用域(静态作用域): 是作用域的分类!
Js中的作用域就属于词法作用域
函数中变量的访问,只和函数声明的位置有关系,也就是和代码的声明书写结构有关系,和函数的具体调用,运行时的状态没有关系,这种作用域就是词法作用域

和词法作用域相对的作用域叫做动态作用域

作用域链

函数可以创建作用域,函数中可以声明函数,这样就形成了作用域嵌套作用域的一个链式的访问结构,称作作用域链

变量的访问规则

当使用一个变量的时候,遵守如下规则

  1. 先在使用变量的作用域中查找,如果找到该变量的声明,就直接使用该变量
  2. 如果没有找到就去上一级作用域中进行查找,如果找到了该变量的声明,就直接使用该变量
  3. 如果没有找到,就继续沿着作用域链向外查找，直到找到全局作用域为止

变量提升

  1. JS代码执行是分阶段的
    1.1 预解析阶段
    1.2 执行阶段
  2. 变量提升
    在js代码的预解析阶段，系统会将所有的变量的声明以及函数的声明提升，提升到其所在的作用域的最顶上

变量提升的特殊情况

  1. 函数同名情况(后面覆盖前面)
    当在变量提升的时候，遇到函数同名的情况的时候，所有的函数都会被提升，但是后面的会把前面的给覆盖掉！

  2. 函数和变量同名的情况(只提升函数忽略变量)
    当函数和变量同名的时候，只提升函数，忽略掉变量声明

  3. 变量提升是分作用域的

  4. 变量提升是分段的(有两组script标签)

  5. 条件式函数声明
    (在条件语句中声明的函数,就是条件式函数声明)
    条件式函数声明在提升的时候,会被当做函数表达式来处理
    提升后的效果就是,只有函数名被提升了,函数体没有被提升

  6. 函数形参在变量提升过程中的处理

在进行变量提升之前,形参的声明以及赋值其实已经完成
所以形参的声明和赋值不参与变量提升

在做变量提升的时候,依次去找变量以及函数的声明,将其放在作用域的最顶上
