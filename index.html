1.JSON格式

  1. 复合类型的值只能是数组或对象，不能是函数、正则表达式对象、日期对象。

  2. 简单类型的值只有四种：字符串、数值（必须以十进制表示）、布尔值和null（不能使用NaN, Infinity, -Infinity和undefined）。

  3. 字符串必须使用双引号表示，不能使用单引号。

  4. 对象的键名必须放在双引号里面。

  5. 数组或对象最后一个成员的后面，不能加逗号。

2.JSON.stringify()

2.1 基本用法

JSON.stringify方法用于将一个值转为字符串。该字符串符合 JSON 格式，并且可以被JSON.parse方法还原。

  1. 需要注意的是，对于原始类型的字符串，转换结果会带双引号。

  1. 如果原始对象中，有一个成员的值是undefined、函数或 XML 对象，这个成员会被过滤。

  2. 如果数组的成员是undefined、函数或 XML 对象，则这些值被转成null。

  3. 正则对象会被转成空对象。

  4. JSON.stringify方法会忽略对象的不可遍历属性

2.2第二个参数

  1. JSON.stringify方法还可以接受一个数组，作为第二个参数，指定需要转成字符串的属性。

第二个参数还可以是一个函数，用来更改JSON.stringify的默认行为。

2.3第三个参数

JSON.stringify还可以接受第三个参数，用于增加返回的JSON字符串的可读性。如果是数字，表示每个属性前面添加的空格（最多不超过10个）；如果是字符串（不超过10个字符），则该字符串会添加在每行前面。

2.4toJSON方法

如果对象有自定义的toJSON方法，那么JSON.stringify会使用这个方法的返回值作为参数，而忽略原对象的其他属性。
toJSON方法的一个应用是，将正则对象自动转为字符串。因为JSON.stringify默认不能转换正则对象，但是设置了toJSON方法以后，就可以转换正则对象了。

JSON.parse()

JSON.parse方法用于将JSON字符串转化成对象。

JSON.parse方法可以接受一个处理函数，用法与JSON.stringify方法类似。

arguments对象

arguments对象就是一个伪数组，他是函数中的一个变量
当函数被调用的时候，传入函数的所有的实参，都会被存入arguments这个伪数组中

一般在用户传入的参数个数不确定的情况下，就使用arguments来获取用户传入的参数，而不通过形参来获取

arguments对象的属性
callee: 指的就是arguments对象所在的函数! 一般用在函数中实现递归

length: 指的是用户在调用函数的时候,传入的实参的个数

递归(算法)

在函数中,直接或者间接的调用自己

递归的两大要素

  1. 必须要有自己调用自己
  2. 必须要有结束条件

化归思想

化归思想，将一个问题由难化易，由繁化简，由复杂化简单的过程称为化归，它是转化和归结的简称。

词法作用域

作用域

变量起作用的范围,就是变量的作用域

在JS当中有且只有函数可以创建作用域

全局作用域：直接写在script标签中的内容，就属于全局作用域的范围
局部作用局：写在函数中的内容，属于局部作用域的范围
块级作用域: （JS中没有块级作用域）通过代码块限定的作用域，就是块级作用域

词法作用域(静态作用域): 是作用域的分类!
Js中的作用域就属于词法作用域
函数中变量的访问,只和函数声明的位置有关系,也就是和代码的声明书写结构有关系,和函数的具体调用,运行时的状态没有关系,这种作用域就是词法作用域

和词法作用域相对的作用域叫做动态作用域

作用域链

函数可以创建作用域,函数中可以声明函数,这样就形成了作用域嵌套作用域的一个链式的访问结构,称作作用域链

变量的访问规则

当使用一个变量的时候,遵守如下规则

  1. 先在使用变量的作用域中查找,如果找到该变量的声明,就直接使用该变量
  2. 如果没有找到就去上一级作用域中进行查找,如果找到了该变量的声明,就直接使用该变量
  3. 如果没有找到,就继续沿着作用域链向外查找，直到找到全局作用域为止

变量提升

  1. JS代码执行是分阶段的
    1.1 预解析阶段
    1.2 执行阶段
  2. 变量提升
    在js代码的预解析阶段，系统会将所有的变量的声明以及函数的声明提升，提升到其所在的作用域的最顶上

变量提升的特殊情况

  1. 函数同名情况(后面覆盖前面)
    当在变量提升的时候，遇到函数同名的情况的时候，所有的函数都会被提升，但是后面的会把前面的给覆盖掉！

  2. 函数和变量同名的情况(只提升函数忽略变量)
    当函数和变量同名的时候，只提升函数，忽略掉变量声明

  3. 变量提升是分作用域的

  4. 变量提升是分段的(有两组script标签)

  5. 条件式函数声明
    (在条件语句中声明的函数,就是条件式函数声明)
    条件式函数声明在提升的时候,会被当做函数表达式来处理
    提升后的效果就是,只有函数名被提升了,函数体没有被提升

  6. 函数形参在变量提升过程中的处理

在进行变量提升之前,形参的声明以及赋值其实已经完成
所以形参的声明和赋值不参与变量提升

在做变量提升的时候,依次去找变量以及函数的声明,将其放在作用域的最顶上

构造函数的相关概念

构造函数是什么

构造函数就是一个函数,构造函数一般被用来初始化对象

构造函数的特点

  1. 一般情况下,首字母大写
  2. 一般情况下,和new 关键字一起使用
  3. 构造函数不需要自己手动去写return语句,会自动返回创建出来的对象

构造函数的执行过程

  1. new关键字创建对象
  2. 调用构造函数,将构造函数内部的this指向new创建出来的对象
  3. 执行构造函数的代码,通过this为刚才创建出来的对象新增属性和方法
  4. 默认自动返回了刚才创建出来并且已经田家湾属性和方法的对象

构造函数的注意事项

  1. 如果手动写了return语句
    1.1 如果return的是值类型的数据,name不会对默认返回值产生影响
    1.2 如果return的是引用类型的数据,那么返回的是就是return语句后面的数据
  2. 如果把构造函数当做普通函数来调用
    2.1 函数中的this会指向window,通过this添加的所有的属性和方法全部添加给了window
    2.2 此时构造函数的返回值,完全按照普通函数来理解

传统的构造函数写法存在的问题

  1. 如果将方法的声明放在构造函数里面,name每次创建对象方法声明都会被执行一次,生成一个新的方法
  2. 每个对象都拥有一个独立的这个方法,但是,所有对象拥有的方法功能都是完全一致的,这样就造成了资源浪费

值类型和引用类型

  1. 值类型:
    直接存储的就是数据本身的这种数据,就是值类型的数据
  2. 引用类型
    存储的是数据的地址,数据单独存起来的这种数据,就是引用类型的数据

特征

  • 当值类型的数据进行赋值的时候,会将现有的变量存储的数据直接复制一份交给新的变量,此时两个变量虽然纸一样,但是存储的空间完全独立,修改其中一个,另一个不受影响

  • 引用类型的数据在赋值的时候,会将变量中存储的对象的地址复制一份交给新的变量,此时两个变量同时指向了同一个对象,修改其中一个,另外一个也会受影响

  • 简单来说 值类型赋值的数据,引用类型赋值的是数据的存储地址

如果在调用writeHead() 方法之前调用了 write() 或者end()方法,系统会自动帮你调用writeHead()方法,并且会生成默认的响应头

如果通过res,setHeader()也设置了响应头,那么系统会将setHeader()设置响应头和writeHeader()设置的响应头合并,并且writeHead()的设置优先

res.write(chunk[,encoding][,callback])

参数1: 要写入的数据,可以是字符串或者二进制数据
参数2: 编码,默认是utf8,选填
参数3: 回调函数,选填


res.end([data][,encoding][,callback])
结束响应

res.end()每次响应都要调用这个方法,用来结束响应

参数1: 结束响应前要发送的数据
参数2: 编码
参数3: 回调函数

res.setHeader(name,value)
* 设置响应报文头

res.statusCode
* 设置或读取http响应码

res.statusMessage
* 设置读取http响应状态的消息

npm  是包管理工具

在项目的根目录下，执行npm install 包名称安装
通过npm install 包名安装的包，会自动下载到当前目录下的node_modules目录下，如果该目录不存在，则创建，如果已存在则直接下载进去。 

npm的全局安装

npm install 包名 -g   
npm全局安装指的是把包安装成了一个命令行工具

install 安装包
uninstall 卸载包  npm uninstall 包名
version  npm version 或者 npm -v

npm init 创建一个package.json 文件 一般用npm init -y

Module模块 和package 包的区别

模块可以是任何一个文件或目录(目录下可以有很多文件),只要能被node.js通过require()即可.

包是一个文件或者目录(目录下可以有多个文件),必须有一个package.json 文件来描述,就可以是一个包

package.json

是一个包说明文件(项目描述文件),用来管理组织一个包(一个项目)
是一个json格式的文件
位于当前项目的根目录下

package.json 文件中常见的项有哪些？

  • name
    • 包的名字
  • version
    • 包的版本
  • description
    • 包描述
  • author
    • 包的作者
  • main
    • 包的入口文件,从main字段这里指定的那个js文件开始执行
  • dependencies
    • 当前包依赖的其他包
